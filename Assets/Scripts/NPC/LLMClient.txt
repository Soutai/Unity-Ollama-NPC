[1. 功能概要]
--------------------------------------------------------------------------------
本类负责与本地 Ollama 服务进行通信，为 NPC 提供文本生成能力。 
包含以下核心功能：
- 通信管理：通过 HTTP POST 请求与本地大模型接口交互。 
- 异步请求：支持标准的一次性非流式请求（Non-streaming）。 
- 流式输出：支持流式请求（Streaming），并能控制字符显示的“打字机”速度。 
- 数据解析：对 JSON 响应及流式数据块进行解析与字符转义处理。 

[2. 变量定义 (Constants & Variables)]
--------------------------------------------------------------------------------
■ 变量名：ollamaUrl
  - 类型：string
  - 权限：private
  - 默认值："http://localhost:11434/api/generate"
  - 说明：本地 Ollama API 的访问地址。 

■ 变量名：charactersPerSecond
  - 类型：float
  - 权限：public
  - 默认值：30f
  - 说明：控制流式输出时每秒显示的字符数量，用于调节打字速度。 

■ 内部类 (Data Models)
  - OllamaResponse：用于接收非流式请求的完整 JSON 响应。 
  - OllamaStreamChunk：用于接收流式请求中单个 Token 的数据块。 

[3. 方法逻辑 (Methods)]
--------------------------------------------------------------------------------
■ 方法：AskAI(string prompt, Action<string> callback)
  - 【逻辑概括】：外部调用的简易接口，发起一次性 AI 文本请求。 
  - 执行逻辑：开启 PostRequest 协程处理网络请求。 

■ 方法：AskAIStream(string prompt, Action<string> onTokenReceived, Action<string> onComplete)
  - 【逻辑概括】：推荐用于 NPC 对话的流式接口，实时返回生成的文本片段。 
  - 执行逻辑：开启 StreamRequest 协程处理分块数据接收。 

■ 核心协程：PostRequest(string prompt, Action<string> callback)
  - 【逻辑概括】：执行标准的 HTTP POST 请求并处理结果。
  - 执行逻辑：
    1. 【JSON 封装】：将模型名称（qwen2:1.5b）、提示词及 stream:false 封装为 JSON 字节流。 
    2. 【网络传输】：使用 UnityWebRequest 发送请求并等待返回。 
    3. 【回调处理】：成功时通过 JsonUtility 解析结果并执行回调；失败时输出错误并返回“连接失败”提示。 

■ 核心协程：StreamRequest(string prompt, Action<string> onTokenReceived, Action<string> onComplete)
  - 【逻辑概括】：执行流式请求并逐行解析数据。
  - 执行逻辑：
    1. 【发起请求】：配置 stream:true 模式发起 POST 请求。 
    2. 【数据分拆】：将返回的文本按行（\n）进行分割处理。 
    3. 【逐块解析】：
       - 遍历每一行数据，反序列化为 OllamaStreamChunk 对象。 
       - 提取 response 字段并通过 onTokenReceived 回调实时更新 UI。 
       - 检测 done 标志位，若为 true 则触发 onComplete 结束回调。 
    4. 【速度控制】：根据 charactersPerSecond 计算间隔时间（1/fps），实现平滑输出。 

■ 方法：EscapeJson(string str)
  - 【逻辑概括】：对字符串中的双引号和换行符进行转义，确保 JSON 格式合法。 

[4. 依赖关系]
--------------------------------------------------------------------------------
- 命名空间：UnityEngine.Networking (处理网络请求), System.Collections (协程支持)。 
- 外部环境：需要本地运行 Ollama 并在指定端口（11434）开启服务，且模型库中包含 qwen2:1.5b。 
- 协作对象：通常被 NPC 决策逻辑调用，用于生成 NPC 的内心独白或对话文本。
