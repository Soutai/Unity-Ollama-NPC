【详细设计书】NPCAnimationBridge.cs

版本：1.0
模块：NPC 系统 - 导航速度与动画状态机桥接
文件：NPCAnimationBridge.cs 
--------------------------------------------------------------------------------

[1. 功能概要]
--------------------------------------------------------------------------------
本类作为 NavMeshAgent 与 Animator 之间的中间层，负责将物理移动参数转化为动画表现。 
包含以下核心功能：
- 自动提取 NavMeshAgent 的速度矢量，计算水平面上的实际移动速率。 
- 方向镜像控制：根据 X 轴速度自动翻转 SpriteRenderer。 
- 动画参数映射：将方向与平滑后的速度值同步至 Animator 的混合树（Blend Tree）。 
- 运动层级转换：根据速度阈值在“行走”与“奔跑”动画状态间平滑过渡。 

[2. 变量定义 (Constants & Variables)]
--------------------------------------------------------------------------------
■ 变量名：acceleration
  - 类型：float
  - 权限：public
  - 默认：10f
  - 说明：动画速度参数改变时的平滑度，防止动画在静止和移动间瞬间切换。 

■ 变量名：currentAnimSpeed
  - 类型：float
  - 权限：private
  - 默认：0f
  - 说明：当前应用于 Animator "Speed" 参数的内部平滑数值。 

■ 私有组件引用：
  - agent (NavMeshAgent): 核心组件，提供实时速度 [velocity] 数据。 
  - animator (Animator): 执行动画状态切换与参数更新的目标组件。 
  - spriteRenderer (SpriteRenderer): 负责 2D 贴图显示及左右镜像翻转。 

[3. 方法逻辑 (Methods)]
--------------------------------------------------------------------------------
■ 方法：Start() [Unity 回调]
  - 【逻辑概括】：获取必要的组件引用并进行安全性自检。
  - 执行逻辑：
    1. 【获取引用】：从自身或子物体中尝试获取 Animator 和 SpriteRenderer 实例。 
    2. 【调试检查】：若关键组件（animator/spriteRenderer）为空，则输出警告日志以便开发者排查。 

■ 方法：Update() [Unity 回调]
  - 【逻辑概括】：每帧执行动画同步逻辑。
  - 执行逻辑：
    1. 【安全检查】：若 [agent] 或 [animator] 丢失，直接跳过逻辑以避免空指针报错。 
    2. 【执行】：调用 [UpdateAnimation] 方法更新当前帧的动画状态。 

■ 方法：UpdateAnimation()
  - 【逻辑概括】：核心同步逻辑，将物理位移信息映射为 2D 动画参数。
  - 执行逻辑：
    1. 【速度计算】：提取 agent.velocity 的 X 与 Z 分量，计算出地面水平速度 [groundSpeed]。 
    2. 【运动分支处理】：
       - **若处于移动状态 (groundSpeed > 0.1f)**：
         - 【镜像翻转】：若 X 轴方向分量大于 0.1，根据其正负设置 spriteRenderer.flipX（向左为 true，向右为 false）。 
         - 【参数映射】：将归一化的方向矢量 X 与 Z 分别传递给 Animator 的 "Horizontal" 与 "Vertical" 参数。 
         - 【阈值判断】：根据 agent.speed 是否大于 3.0f 来决定目标动画强度（2.0f 为跑，1.0f 为走）。 
       - **若处于静止状态**：
         - 将 [targetValue] 设为 0。 
    3. 【平滑处理】：使用 Mathf.MoveTowards 配合 [acceleration] 将 [currentAnimSpeed] 逼近目标值。 
    4. 【结果更新】：调用 animator.SetFloat 更新最终的 "Speed" 混合树参数。 

[4. 依赖关系]
--------------------------------------------------------------------------------
- 强制依赖：同一 GameObject 必须包含 NavMeshAgent 组件。 
- 结构要求：Animator 和 SpriteRenderer 需位于自身或层级结构的子物体中。 
- 协作对象：依赖 Animator 控制器内部定义的参数（Speed, Horizontal, Vertical）。
