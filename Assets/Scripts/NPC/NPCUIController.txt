--------------------------------------------------------------------------------

[1. 功能概要]
--------------------------------------------------------------------------------
本类负责将 NPC 的内部逻辑状态（生理需求、思考决策）同步到场景中的 UI 元素上。 
包含以下核心功能：
- 状态可视化：将饥饿值同步到 Slider 血条，并根据数值动态切换颜色（红/黄/绿）。 
- 内心独白显示：将 AI 大脑当前的思考逻辑实时更新至 TextMeshPro 文本框。 
- 健壮性保障：自动寻找父级核心组件，并具备完善的组件缺失防御逻辑。 

[2. 变量定义 (Constants & Variables)]
--------------------------------------------------------------------------------
■ 外部 UI 槽位 (Serialized References)
  - hungerSlider (Slider): 用于显示饥饿程度的进度条组件。 
  - thoughtText (TextMeshProUGUI): 用于显示 NPC 思考内容的文本组件。 
  - canvasGroup (CanvasGroup): 用于控制 UI 整体透明度或交互开关的组件。 

■ 私有逻辑引用 (Internal Logic)
  - needs (NPCNeeds): 提供饥饿值等原始生理数据。 
  - brain (NPCUtilityBrain): 提供当前的思考状态字符串。 

[3. 方法逻辑 (Methods)]
--------------------------------------------------------------------------------
■ 方法：Start() [Unity 回调]
  - 【逻辑概括】：初始化数据链接，配置 UI 基础极值。
  - 执行逻辑：
    1. 【组件查找】：使用 GetComponentInParent 在父物体中搜索 NPCNeeds 和 NPCUtilityBrain。 
    2. 【容错检查】：若找不到核心组件，则抛出错误或警告日志，防止后续引用空指针。 
    3. 【UI 初始化】：若 Slider 存在，强制设置其最大值为 100f，最小值为 0f。 

■ 方法：Update() [Unity 回调]
  - 【逻辑概括】：每帧同步数据。
  - 执行逻辑：
    1. 【核心防御】：若 [needs] 引用缺失，直接中断后续逻辑执行。 
    2. 【数据同步】：
       - 若 Slider 不为空：将 [needs.hunger] 映射至 Slider 数值，并调用 [UpdateSliderColor]。 
       - 若 Text 和 Brain 同时存在：将 [brain.currentThought] 同步给文本显示。 

■ 方法：UpdateSliderColor()
  - 【逻辑概括】：根据饥饿程度提供视觉预警。
  - 执行逻辑：
    1. 获取 Slider 填充区域（fillRect）的 Image 组件。 
    2. 【颜色阈值判定】：
       - [hunger] < 25f：设为红色（紧急预警）。 
       - [hunger] < 60f：设为黄色（中度饥饿）。 
       - 其他情况：设为绿色（状态良好）。 

[4. 依赖关系]
--------------------------------------------------------------------------------
- 依赖项：UnityEngine.UI, TMPro 
- 层级要求：本脚本通常挂载在 NPC 的 UI 子物体上，核心逻辑脚本（Needs/Brain）需位于父物体中。 
- 协作对象：
    - NPCNeeds: 提供 [hunger] 数据源。 
    - NPCUtilityBrain: 提供 [currentThought] 文本源。
