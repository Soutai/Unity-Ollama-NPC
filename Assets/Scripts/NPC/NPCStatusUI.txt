【详细设计书】NPCStatusUI.cs
================================================================================
版本：1.0
模块：UI 系统 - NPC 状态可视化监控
文件：NPCStatusUI.cs
--------------------------------------------------------------------------------

[1. 功能概要]
--------------------------------------------------------------------------------
本类负责将 NPC 的内部生理数值与决策状态实时映射到 UI 界面上[cite: 23]。
包含以下核心功能：
- 饥饿条视觉反馈：同步显示饥饿值的比例进度[cite: 23]。
- 动态颜色预警：根据数值剩余量自动切换 UI 颜色（健康/警告/危险）[cite: 23]。
- 文本同步更新：显示具体的饥饿百分比及 NPC 当前正在执行的动作意图[cite: 23]。

[2. 变量定义 (Constants & Variables)]
--------------------------------------------------------------------------------
■ 绑定 UI 组件 (TextMeshPro & Image)：
  - statusText (TextMeshProUGUI): 用于显示 NPC 当前行为描述的文本框[cite: 23]。
  - hungerFillImage (Image): 饥饿条的填充图像，要求 Image Type 为 Filled[cite: 23]。
  - hungerPercentText (TextMeshProUGUI): 用于显示“XX%”格式的数字文本[cite: 23]。

■ 颜色设置 (Color)：
  - normalColor: 饥饿值充足时的颜色（默认绿色）[cite: 23]。
  - warningColor: 饥饿值进入警戒线时的颜色（默认黄色）[cite: 23]。
  - criticalColor: 饥饿值极低时的颜色（默认红色）[cite: 23]。

■ 追踪目标：
  - targetNPC (NPCNeeds): 指向需要被监控的 NPC 实例引用[cite: 23]。

[3. 方法逻辑 (Methods)]
--------------------------------------------------------------------------------
■ 方法：Update() [Unity 回调]
  - 【逻辑概括】：每帧轮询目标 NPC 的最新数据并刷新 UI 表现[cite: 23]。
  - 执行逻辑：
    1. 【有效性检查】：确保 [targetNPC] 不为空[cite: 23]。
    2. 【比例计算】：读取 targetNPC.hunger 并归一化为 0.0 - 1.0 的浮点数[cite: 23]。
    3. 【进度更新】：更新 hungerFillImage.fillAmount 并调用颜色更新逻辑[cite: 23]。
    4. 【文本更新】：将比例转换为 0 位小数的百分比字符串[cite: 23]，并同步显示 targetNPC.currentAction[cite: 23]。

■ 方法：UpdateBarColor(float fill)
  - 【逻辑概括】：基于剩余比例动态调整 UI 组件的色彩[cite: 23]。
  - 权限：private
  - 参数：fill (float) - 当前饥饿值的比例值 (0-1)[cite: 23]。
  - 执行逻辑：
    1. 【危险判定】：若 fill <= 0.3 (30% 以下)，使用 criticalColor[cite: 23]。
    2. 【警告判定】：若 fill <= 0.6 (60% 以下)，使用 warningColor[cite: 23]。
    3. 【健康判定】：其他情况使用 normalColor[cite: 23]。

[4. 依赖关系]
--------------------------------------------------------------------------------
- 命名空间：UnityEngine, UnityEngine.UI, TMPro[cite: 23]。
- 外部依赖：
    - NPCNeeds: 提供 hunger 数值及 currentAction 文本[cite: 23]。
- UI 要求：
    - Canvas 系统：必须放置在 Canvas 下方。
    - 组件配置：填充图 (hungerFillImage) 必须在 Inspector 中将填充模式设为 Horizontal 或 Radial[cite: 23]。
