文件：HungerBarUI.cs 
--------------------------------------------------------------------------------

[1. 功能概要]
--------------------------------------------------------------------------------
本类负责实时更新玩家饥饿值的 UI 表现，将数值逻辑转化为视觉进度条。 
包含以下核心功能：
- 组件自动关联：初始化时自动检索玩家引用及数值系统。 
- 实时百分比计算：基于当前饥饿值与最大值的比例计算填充率。 
- 视觉映射：动态同步 UI Image 组件的填充属性。 

[2. 变量定义 (Constants & Variables)]
--------------------------------------------------------------------------------
■ 变量名：hungerFillImage
  - 类型：Image
  - 权限：public
  - 说明：指向 UI 进度条的填充层图片组件。 

■ 变量名：playerHunger
  - 类型：HungerSystem
  - 权限：private
  - 说明：缓存玩家身上的饥饿系统引用，用于读取实时数值。 

[3. 方法逻辑 (Methods)]
--------------------------------------------------------------------------------
■ 方法：Start() [Unity 回调]
  - 【逻辑概括】：执行场景检索以建立数据链路。 
  - 执行逻辑：
    1. 【查找】：通过 Tag 寻找场景中标记为 "Player" 的游戏对象。 
    2. 【组件获取】：若对象存在，获取其挂载的 [HungerSystem] 组件并存入 [playerHunger]。 

■ 方法：Update() [Unity 回调]
  - 【逻辑概括】：每帧刷新进度条的显示状态。 
  - 执行时机：每一帧 (Every Frame) 
  - 执行逻辑：
    1. 【安全判定】：检查 [playerHunger] 数据源与 [hungerFillImage] 目标组件是否均已就绪。 
    2. 【计算】：使用公式 (currentHunger / maxHunger) 得出 0 到 1 之间的浮点比例。 
    3. 【更新】：将计算结果赋值给 [hungerFillImage.fillAmount]。 

[4. 依赖关系]
--------------------------------------------------------------------------------
- 依赖项：UnityEngine.UI (必须引用 UI 命名空间)。 
- 协作对象：
    - HungerSystem: 提供基础的饥饿数值支持。
- 场景要求：
    - 玩家物体必须带有 "Player" 标签。 
    - UI Image 组件的 "Image Type" 属性必须设置为 "Filled" 模式。
