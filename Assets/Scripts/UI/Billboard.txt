[1. 功能概要]
--------------------------------------------------------------------------------
本类用于实现 2D 元素在 3D 空间中始终面向摄像机的效果，常用于角色 Sprite 或 UI 气泡。 
包含以下核心功能：
- 实时朝向同步：确保物体平面始终平行于摄像机视平面。 
- 渲染层级优化：在 LateUpdate 中执行，避免摄像机移动带来的抖动。 
- 镜像修复逻辑：自动抵消由于父物体 Scale 值为负导致的非预期镜像翻转问题。 

[2. 变量定义 (Constants & Variables)]
--------------------------------------------------------------------------------
■ 变量名：mainCamera
  - 类型：Camera
  - 权限：private
  - 说明：缓存场景中的主摄像机引用，用于计算朝向矢量。 

[3. 方法逻辑 (Methods)]
--------------------------------------------------------------------------------
■ 方法：Start() [Unity 回调]
  - 【逻辑概括】：初始化时自动获取主摄像机。
  - 执行逻辑：
    1. 【缓存引用】：通过 Camera.main 获取并存储当前主摄像机的实例。 

■ 方法：LateUpdate() [Unity 回调]
  - 【逻辑概括】：在每一帧渲染前更新物体的旋转与缩放状态。
  - 执行逻辑：
    1. 【安全检查】：若 [mainCamera] 为空则直接跳过，防止空指针异常。 
    2. 【面向相机】：使用 LookAt 方法，结合摄像机的 Forward 和 Up 矢量，使物体始终正对相机视线。 
    3. 【镜像补偿】：
       - 检测 [lossyScale]（全局缩放）是否存在负值。 
       - 若检测到负值，判断父物体的 X 轴缩放情况。 
       - 动态修正 [localScale]，通过正负抵消逻辑确保 Sprite 不会因为父级的镜像操作而反向显示。 

[4. 依赖关系]
--------------------------------------------------------------------------------
- 依赖项：UnityEngine 
- 协作对象：
    - NPCSpeech: 用于 NPC 对话气泡，确保文字始终可见。
    - PlayerMovement3D: 用于 2D 角色 Sprite 在 3D 场景中的视觉呈现。
- 场景要求：场景中必须存在 Tag 为 "MainCamera" 的摄像机。
