【详细设计书】MinimapSync.cs

版本：1.0
模块：基础系统 - 小地图相机与导航同步
文件：MinimapSync.cs 
--------------------------------------------------------------------------------

[1. 功能概要]
--------------------------------------------------------------------------------
本类负责小地图摄像机的位置同步，以及玩家图标（如箭头）在地图上的指向逻辑。 
包含以下核心功能：
- 相机平滑跟随：使小地图相机保持在玩家正上方并固定高度。 
- 动态指向逻辑：根据玩家的实时物理速度（Velocity）计算并旋转图标方向。 
- 静态标记支持：提供 UI 层面北方标志（North Text）的绑定引用。 

[2. 变量定义 (Constants & Variables)]
--------------------------------------------------------------------------------
■ 绑定对象 (Binding Objects)
  - player (Transform): 指向场景中的玩家主体物体。 
  - arrowIcon (Transform): 位于小地图层级的玩家方向图标。 
  - northText (RectTransform): UI 上的北方方位参考标识。 

■ 配置项 (Settings)
  - cameraHeight (float): 小地图相机距离地面的固定 Y 轴高度，默认 20.0f。 

■ 私有组件 (Internal Components)
  - playerController (CharacterController): 缓存玩家的物理控制器，用于读取速度数据。 

[3. 方法逻辑 (Methods)]
--------------------------------------------------------------------------------
■ 方法：Start() [Unity 回调]
  - 【逻辑概括】：初始化并建立物理组件的关联。
  - 执行逻辑：
    1. 【组件获取】：检查 [player] 是否存在，若存在则获取其挂载的 [CharacterController]。 

■ 方法：LateUpdate() [Unity 回调]
  - 【逻辑概括】：在玩家移动更新后同步相机位置与图标角度，防止画面抖动。
  - 执行时机：每一帧 (Late Update)。 
  - 执行逻辑：
    1. 【判空检查】：若 player 或 arrowIcon 为空，则终止执行。 
    2. 【相机跟随】：将当前物体的坐标 X 和 Z 与玩家对齐，Y 轴设为 [cameraHeight]。 
    3. 【旋转计算】：
       - 读取 [playerController.velocity] 获取移动向量。 
       - 【条件判断】：仅当速度大小（magnitude）大于 0.1f 时执行旋转，避免停止时方向重置。 
       - 【数学转换】：利用 Mathf.Atan2(x, z) 将 XZ 平面的分量转为弧度，并换算为角度。 
    4. 【应用旋转】：设置 [arrowIcon.rotation] 为欧拉角 (90, targetAngle, 0)，确保图标平贴地面。 

[4. 依赖关系]
--------------------------------------------------------------------------------
- 依赖项：UnityEngine 
- 组件要求：[player] 对象必须挂载 CharacterController 才能实现动态指向。 
- 层级要求：[arrowIcon] 通常应设置在特定的 Minimap Layer 上，仅供小地图相机渲染。 
- 协作对象：此脚本挂载在专门的小地图渲染相机（Minimap Camera）上。
