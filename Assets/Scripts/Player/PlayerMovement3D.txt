【详细设计书】PlayerMovement3D.cs

版本：1.0
模块：玩家控制系统 - 3D 移动与动画同步
文件：PlayerMovement3D.cs 
--------------------------------------------------------------------------------

[1. 功能概要]
--------------------------------------------------------------------------------
本类实现了玩家角色的双模移动控制（键盘 + 鼠标点击）以及配套的 2D 序列帧动画同步。 
包含以下核心功能：
- 混合移动模式：支持 WASD 键盘输入，并兼容鼠标点击地面导航。 
- 速度切换：支持通过 LeftShift 键在“行走”与“奔跑”状态间切换。 
- 动画状态映射：将 3D 位移矢量转化为 2D 混合树参数，并处理 Sprite 的水平翻转。 
- 物理锁定：强制约束 Y 轴坐标，防止 3D 物理计算导致的角色浮空或下陷。 

[2. 变量定义 (Constants & Variables)]
--------------------------------------------------------------------------------
■ 移动配置 (Movement Settings)
  - walkSpeed (float): 基础步行速度，默认 5.0f。 
  - runSpeed (float): 按住加速键时的奔跑速度，默认 9.0f。 
  - acceleration (float): 动画参数平滑切换的速率，影响动作启停的视觉手感。 
  - runKey (KeyCode): 跑步切换键，默认为左 Shift。 

■ 私有组件与状态 (Internal Components & States)
  - controller (CharacterController): 处理 3D 碰撞与位移的核心组件。 
  - animator (Animator): 位于子物体，负责 2D Billboard 动画驱动。 
  - spriteRenderer (SpriteRenderer): 负责角色贴图渲染与镜像控制。 
  - targetPosition (Vector3): 鼠标点击模式下的目标路点坐标。 
  - isMouseMoving (bool): 当前是否处于鼠标驱动的自动行走状态。 

[3. 方法逻辑 (Methods)]
--------------------------------------------------------------------------------
■ 方法：Update() [Unity 回调]
  - 【逻辑概括】：每帧监听输入并计算最终移动矢量。
  - 执行逻辑：
    1. 【键盘采样】：获取 Horizontal/Vertical 轴向输入，归一化得到 [keyboardDir]。 
    2. 【模式判定】：键盘输入优先级高于鼠标。若检测到按键，则打断鼠标移动。 
    3. 【逻辑分支】：
       - 若键盘无输入且 [isMouseMoving] 为真，计算向 [targetPosition] 的指向矢量。 
       - 到达目标点（距离 < 0.2f）后自动停止。 
    4. 【执行】：调用移动与动画处理方法，最后执行 [LockYAxis]。 

■ 方法：UpdateMouseTarget()
  - 【逻辑概括】：通过射线检测将屏幕点击坐标映射至 3D 空间。
  - 执行逻辑：
    1. 从主相机发出射线。若击中物体，检查其标签。 
    2. 【过滤逻辑】：若击中物体是树木（"Tree"）或玩家自身，则不更新目标点。 
    3. 否则，更新 [targetPosition] 并激活 [isMouseMoving]。 

■ 方法：UpdateAnimation(Vector3 dir, bool isRunning)
  - 【逻辑概括】：驱动动画状态机，处理 2D 角色朝向。
  - 执行逻辑：
    1. 【镜像翻转】：若 X 轴分量不为 0，根据正负设置 [spriteRenderer.flipX] 实现左右转向。 
    2. 【参数同步】：将方向向量传递给动画机的 "Horizontal" 与 "Vertical" 参数。 
    3. 【速度平滑】：
       - 移动中：[currentAnimSpeed] 向 1.0f (走) 或 2.0f (跑) 靠近。 
       - 静止中：[currentAnimSpeed] 向 0.0f (Idle) 靠近。 

■ 方法：LockYAxis()
  - 【逻辑概括】：物理层面的强制位置修正，将 Y 轴恒定锁定在 0 位，确保角色始终贴地。 

[4. 依赖关系]
--------------------------------------------------------------------------------
- 强制要求：同一物体需挂载 CharacterController 组件。 
- 结构要求：Animator 和 SpriteRenderer 需位于子物体上，以支持 2D 序列帧展示。 
- 输入系统：依赖 Unity 标准的 Input 管理器（Horizontal/Vertical 轴）。
